# ZEN Study 学習統計（個人用・表示専用）

自分の ZEN Study 学習データを、より見やすく・より深く可視化する個人用 Chrome 拡張。
`/setting` の学習数パネルを高機能カードに置き換え、コース/章画面に残り作業量を表示し、
サイト全体のダークモードも提供する。

> **非公式ツールです。** KADOKAWA・ドワンゴ・N高等学校／S高等学校／R高等学校・ZEN Study 各社/各校とは
> 一切関係のない、いち生徒による個人プロジェクトです。自分のアカウントで自分の学習データを
> **read-only で取得して見やすく表示するだけ**のもので、学習記録の変更や動画進捗の操作は一切行いません。
> 利用は自己責任で、各サービスの利用規約の範囲内でご使用ください。
> [Chrome ウェブストア](https://chromewebstore.google.com/detail/bkmampnickklonnmenhclffblfeojndd)から
> インストールできます（リリースzip / ソースビルドでも可・下記「[導入](#導入)」参照）。
>
> **運営の位置づけ（2026-07 問い合わせ確認）**: ZEN Study 運営はサードパーティ製ツールの個別の検証・許諾を
> 行っておらず、「利用規約の範囲内で自己責任」での利用となります。拡張機能を入れた状態で生じた不具合は
> 運営サポートの対象外です — **本家サイトの不具合を運営に問い合わせる際は、必ず本拡張を無効化して
> 再現確認（切り分け）をしてから**にしてください。

## 第一原則（厳守）
1. **GET のみ。状態変更(POST/PUT/PATCH/DELETE)は一切しない（完全 read-only）**
2. 動画の不正視聴・進捗の自動進行は絶対にしない。学習記録を一切変更しない
3. 表示専用。DOM操作は自分のブラウザの描画のみ
4. **レート配慮**: すべての GET は共通ラッパー(`src/http.ts`)経由で、サーキットブレーカー（10秒40件超で60秒遮断）により、バグ由来のリクエスト暴走を設計レベルで防止（規約12-9）

詳細は `API_REFERENCE.md`（エンドポイント地図・調査結果）/ `DESIGN.md`（設計）。

## 主な機能
- **学習数カード**（`/setting`）: 本家パネルを上書き。本家デザイン完全準拠、a11yデータ表つき。
- **コンパクトな要点＋直下に詳細タブ**（`[推移][予測][教科][分析]`）: 常時表示は要点ストリップ（累計・今日・平均・連続）だけに絞り、大きなグラフを含む詳細をトグルで隠さず**直下にタブで常時表示**（この拡張の主役）。各タブは遅延描画。
- **学習モード切替（必修 / 必修以外）**: 要点ストリップ直下のトグルで、予測・教科・分析タブの対象を**必修（`service=basic`）と必修以外（`service=advanced`＝選択科目・講座・大学連携）**で切り替え。必修以外は締切の概念が無い自己ペース学習なので、教科タブは**本家「課外授業」準拠の指標**（理解度＝教材の閲覧進捗 `comprehension`・習熟度テスト＝修了判定 `total_count`）で表示、予測タブは**完了見込み**（締切逆算ではなく、理解度＝教材の閲覧進捗の現在ペースから「順当に進めば◯月◯日ごろ完了」を推定。理解度を教科別に日次スナップ蓄積し数日で表示）＋消化状況、分析タブは着手状況の軽量版になる。必修(n_school)は `total_materials` で進捗、advancedは `comprehension`+習熟度テストという**別データモデル**（本家UIと突合して確定）。**学習数（要点ストリップ）と推移タブは全学習の合計＝モード共通**（`learning_amounts` は必修/非必修を分離不可のため。モードトグルは「下タブの切替」であって学習数のフィルタではない旨を注記で明示）。選択中モードはタブ冒頭のバッジで表示。
  - **推移**: 日別の学習数バー（即描画）＋ 記録メタ1行（開始日・日数・最長連続）＋ **「傾向グラフ」セグメント**で `カレンダー / トレンド / 時間帯 / 曜日` を1枚ずつ切替表示（縦積みを回避しコンパクト化）。GitHub風カレンダーヒートマップ・長期トレンド（日/週/月）・時間帯トレンド（**「完了検知 / 受験·平均 / 受験·日別」の3ビュー**。受験·日別は**日付×時刻のパンチカード**＝いつ勉強したかの実記録を分単位でプロット（縦軸は5時はじまりで深夜学習が連続に見える）。受験系ビューは詳細ログ収集後は導入以前の受験時刻からも表示。さらに**動画の視聴時刻を前後の受験アンカー間から補間** — 必修動画は倍速不可・同時視聴不可＝実時間×直列という本家仕様を根拠に、アンカー間の経過時間が動画合計実時間と整合する窓のみ採用し、長い中断を挟んだ窓は捨てる。妥当性チェックは公開研究に準拠: 許容スラックは time-on-task 推定のセッション・タイムアウト規範から**30分**、配置は区間打ち切りデータの**中点補完**（前詰め＝下限補完の早方向バイアスを排除）、採用は**最大誤差±15分以下**の窓のみ（時間帯バケット幅60分に対し概ね正しいバケットに入る）。誤差はパンチカードのツールチップに ±N分 で明示。両ソースは重複期間があるため合算しない）・曜日別リズムを内包。
  - **実績・完了見込みグラフの教科別表示**: 見出しのセレクタで「全体（既定・従来のモンテカルロ帯付き）」と各教科を切替。教科別は **直接観測（coursePassedHist・実線）＋抽出ログ由来の後方外挿（受験アンカーの確定時刻＋補間動画・点線＝導入以前も含む推定）** の実績に、現在ペースの投影線＋必要ラインを重ねる。教科別ペースは coursePassedHist が薄いとき **受験アンカーの直近ペース（長期間で安定・「受験基準」と明示）** にフォールバック。
  - **予測**: 先頭に **次のレポート締切**（月次の締切を主役に。年度末一発でなく毎月の締切に追いつくのが成績上の実態。次の締切まで残りN日・残り章数・締切超過の警告を提示。さらに**教科別ペース×締切の章内訳から「このペースだと間に合わないかもしれない教科」を警告**＝残り・週ペース・完了見込み日数を教科別に判定。月内に複数締切がある場合は累積残りを各締切と突き合わせ最も厳しい判定を採用。ペース未蓄積の教科は「蓄積中」と明示）＋ **今日のデイリークエスト**（締切から逆算した推奨ペースを「今日 N/目標・あとM教材」で提示）＋ その直下に **実績・完了見込みグラフ**（バーンダウン）を配置 ＋ 年度レポート完了予測（下記）。KPIは3点に集約、手法別の見立ては折りたたみ。デイリークエストの「今日の完了数」は**当日始点(dayStart)からの教材消化差分**で算出し、完了検知でライブ更新。
  - **教科**: 全教科の *残/総* を一覧に統合。**上部の「詳細グラフを表示」ボタン**（押すと全章を集計）で、**教科別に色分けした「残り学習量シェア」ドーナツ**（動画実時間＋テスト/レポートの時間換算。換算は3段階で精緻化: ①完了検知の間隔から自動実測した教科別の分/問（n≥3で採用）②残り問題数×目安 ③固定目安。凡例＋シェア%＋スライス間ギャップで色以外でも識別可）と、動画時間・本数・確認テスト・レポートの残/総の内訳（章別＋a11y表）を表示。既定は教材の残/総一覧（クリックでコースへ）＋全体進捗ドーナツ。
  - **分析**: 先頭に **モチベーション「今日のひとこと」**（行動科学の実証手法に基づくナッジ＝下記）＋「あなたの学習傾向」— 必修アドバイス／**レポート締切の状況**（締切遵守率＝**記録開始後に締切をまたいだ月**を期限内に守れた割合。締切前に観測した進捗を締切またぎで凍結して判定するため、導入前・転入前に過ぎていた締切や締切後の駆け込み完了で歪まない。＋締切超過の警告・次の締切と必要ペース）／**トレンド判定**（線形回帰の傾き＋Mann-Kendall検定で「伸びている/横ばい/落ちている」、週モメンタム、ラグ1自己相関で"好日の連鎖"）／**分布サマリー**（中央値＝普段の1日・上位1割＝好調日・0の日・パレート集中度）／曜日リズム（**Kruskal-Wallis検定で差が有意かノイズかを明示**＝小標本での過信防止）／時間帯（完了検知でその時刻を記録）／**教科別ペース**（教科ごとの passed を日次蓄積し、消化ペースと教科別の完了見込みを表示。数日ぶん貯まると出る。「この締切に間に合わない教科」の警告は予測タブの締切カードに表示）／**所要時間の実測**（テスト/レポートの平均 分/本・分/問を教科別に一覧。完了検知の間隔0.5〜45分のみ採用）／月ごと（前月比）／祝日／継続（CV＋学習日率＋最長空白＋バースト度）。冒頭に**「これまでの歩み」**（記録開始からの累計・最長連続・ベスト日・実績数のWrapped風サマリ）。統計関数は `src/stats.ts`（純関数）。
- **どこでも見られるサイドパネル**（`/setting` 以外の全ページ）: 画面右端の「📊 学習統計」ハンドルから、学習カード（4タブ全部）を端からスライド表示。固定配置＋Shadow DOM隔離でページ非依存・開いた時のみデータ取得・ダーク連携。
- **モチベーション「今日のひとこと」**: 行動科学/マーケの実証手法に基づくナッジ — Endowed Progress（積み上げ強調）/ Goal-Gradient（完了間近・ラストスパート）/ **Zeigarnik（やりかけの章＝あと3教材以内で章完了を提示）**/ Fresh Start（週初=日曜・月初の仕切り直し）/ Implementation Intentions（ベスト曜日・時間帯から具体的計画）/ Loss Aversion（ストリーク保護）。**おかえりトースト**（前回の記録からの増分を1日1回だけ提示）。詳細は `src/motivation.ts`。
- **控えめなトースト通知**（左下・数秒で自動消滅）: 全体%の節目達成・今日の必要最低限を達成・5:00の日付更新間近・**週明けの「先週のまとめ」**、を通知（永続dedupで繰り返さない）。**通知履歴**: トーストは消えても、カード下部の「通知履歴」から直近60件を見返せる（devトーストは記録対象外）。
- **週間目標**（予測タブ・今日の目標の下）: 週N教材の目標＋今週の進捗バー（**日曜5:00境界**・義務ペース以上のみ設定可）。日次の凸凹を吸収する中間粒度で、Fresh Start 効果（週明けの仕切り直し）と接続。週の途中で始点が設置された場合（導入直後など）は、日次スナップから**週初時点の passed を復元して始点を自動修復**（小さい方のみ採用＝過大計上しない）。
- **実績バッジ**（分析タブ）: 連続学習・累計教材・全体%・教科完了・学習日数の19種を、**実データに基づく達成のみ**解除式で一覧化（初達成日を記録）。XP/リーグ等の比較・変動報酬系は意図的に不採用。
- **予測の的中率**（予測タブ）: 過去の予測（7/14/28日後の残数バンド）を毎日スナップショットし、後日の実績と突き合わせて「P15〜85帯に収まった割合（理想≈70%）」と楽観/悲観の偏りを表示。予測の自己校正という honest なメタ指標。
- **ツールバーポップアップ**: 拡張アイコンから、ZEN Study を開かずに全体%ドーナツ・今日の目標・連続学習・残教材を確認（クリック時のみGET・計4リクエスト）。
- **所要時間タイマー（オプション・既定ON）**: 未提出の `evaluation_test / essay_test / evaluation_report / essay_report` ページを開くと右下に小さなタイマーを表示。本家は解答途中をキャッシュするため、**セッション単位でなく教材ごとの累計**で計測（離脱時保存・再訪で再開・タブ非表示中は停止）。提出を検知すると累計を「所要時間の実測」へ自動記録（この場合、従来の間隔近似との二重計上はしない）。ウィジェットの ✕ または カード「表示設定」でOFF可。
- **連続学習は「生存中の連続」表記**: 今日まだ学習していなくても、昨日までの連続は途切れていないので前日までの日数を表示（0になるのは丸1日空いてから。Duolingo等と同じ扱い）。
- **詳細ログの抽出（受験記録の遡及復元）**: 受験済みテスト/レポートの結果画面（サーバ埋め込みJSON）から**合否・点数・受験日時**を収集し、**拡張の導入以前も含む実測分析**を表示 — 実測サマリ（期間・活動日・教科別受験数）／アクティブ時間帯（answered_at による分単位の実測）／初回合格率（全体＋教科別）／レポート得点率／月別×教科の受験ペース。カード末尾のフォールドから手動起動。取得は1件ずつ間隔を空け（約2件/秒・1回上限600件・途中保存）、**確定結果は永続キャッシュして二度と再取得しない**（2回目以降は新規完了分のみ）。保存は合否・点数・日時のメタデータのみで、問題文・解答文は保存しない（ToS遵守）。**論述（essay系）も対象**（結果は `result.answered_at` 単一スキーマ・点数は採点後に付くため採点待ちの間は自動再確認）。章またぎのアンカー対（前章のレポート→次章の第1回テスト等）も、コース内の章順で連結して動画補間の窓として評価。分析タブにも同セクションを表示。
- **長期データ**（自前蓄積）: ZEN Studyを開いた日に1回スナップショット＋14日窓マージで蓄積（SWのCookie制約を回避）。
- **完了検知（観測のみ）**: 本家が完了時に送る `PUT …/progress/passed`（動画）/ `POST …/answerings`（テスト等）に加え、**`/v1/material/...` 系の提出パス**（essay系answerings・`material/{type}/{id}/progress`・`exercises/{id}/answers`＝本家バンドルに存在する別系統）も **MAIN world の observer.js が観測**（HAR実測＋バンドル静的解析）。検知後の確定は 1.5秒デバウンス＋間隔を広げる再確認 最大5回（計約30秒カバー）で、サーバ集計ラグに強く・反映は速く。**信頼境界**: postMessage はページJSから偽造可能なため「UI更新トリガー」としてのみ扱い（受信時にオリジン・ID形式を検証）、実記録は GET による passed 増分の照合でのみ確定。確定処理は直列化し二重計上を防止。さらに**フック不全へのフォールバック**として、表示中タブで定期GET照合（observer生存時5分・応答なし時2分間隔）を行い、サイト側の fetch 再代入や他拡張との競合で検知が壊れても追いつく。**我々は一切送信しない**（第一原則）。検知はトリガーに過ぎず、**実際に教材完了数(passed)が増えた時だけ**カウント（不合格の提出は passed 不変→無視）。用途: ①時間帯を"完了したその時刻"で正確に記録（誤帰属を根絶）②節目トースト③コース/章バナーの残りを即更新④**開いている予測/教科タブ（今日の目標・進捗）をライブ再描画**。集計API(passed_materials)の反映ラグに備え、増分0なら数秒間隔で最大3回まで再確認する。
- **データのバックアップ / 復元**: 蓄積した永続データ（学習数・完了レポート・教材消化の履歴／時間帯の学習記録／目標完了日／テーマ設定）を JSON で書き出し・復元。CSV は日付×学習数・完了レポート・教材消化の一覧。復元は既存と統合（履歴は日付マージ・時間帯は大きい方を採用・削除なし）。**再構築可能なキャッシュや、当日限り／端末ごとの運用状態（スナップ済みフラグ・完了検知の基準passed・当日始点・通知dedup）は除外**（復元時の不整合＝当日スナップのスキップや完了差分の破壊を防ぐため）。※読み書きは自分の `chrome.storage.local` のみで ZEN Study へは通信しない。自前蓄積データはAPIから復元不能な一点物のため喪失対策。
- **年度レポート完了予測（北極星）**: 教材消化ペースを基に、**モンテカルロ(1000回)で完了日の分布**を算出。「◯までに完了(85%)・締切に間に合う確率X%」。バーンダウン(P15〜85帯＋完了分布＋必要ライン＋**目標日ライン**)、EWMAペース、曜日/祝日考慮、目標日から必要ペース逆算、**週次計画の .ics 書き出し**（目標日までの週次マイルストーンをカレンダーへ・完全ローカル生成）、手法別の見立て。目標日を設定すると、その完了日への理想ペース線をグラフに追加描画（入力に追従・アニメーション付き）。さらに **「一日の教材数から逆算」**（義務ペース＝締切に間に合う最低ペース以上のみ設定可）で、指定した1日ペースでの完了見込み日を算出し、同じく理想ペース線を描画。締切は必修レポートの最終提出期限を動的取得。
- **少データ（cold-start）対策**: 導入直後は14日窓しか無く推定が不安定・過信になりがち。統計/マーケの定石で補強 — ①**14日窓の即時シード**（自前蓄積が空でも予測可）、②**ベイズ平均縮小**（少数の曜日サンプルを全体平均へ縮小し外れ日で暴れない）、③**事後予測的な母数不確実性**をモンテカルロに注入（区間が小標本ほど正しく広がる）、④**予測の確度バッジ**（データ成熟度を明示）。詳細は `src/shrinkage.ts`。
- **学年ロールオーバー耐性（頑健フォールバック）**: 必修は学年で入替わり `passed`/`total` が不連続に変化する。教材スナップショットに **total も日付別に記録**し、ペースサンプルは「total が変化した」or「passed が減少した」区間を**構造を仮定せず一律に除外**（＝検知ではなく防御）。これにより年度替わりでペース推定が壊れない。
- **コース/章画面の残りサマリ**: 「このコース/この章の残り: 動画NN時間・N本 / 確認テストN / レポートN」をバナー注入。**動画・テスト・レポートを完了するたびに、今見ている章のバナーを"静かに"再取得して差し替え**（既存表示を消さず数値だけ更新＝「集計中…」の点滅なし・400msデバウンスで連続完了も1回に集約）。**動画再生中の横メニュー等、内部スクロールコンテナがある画面ではバナーをそのコンテナ上部に `sticky` で張り付かせ**、スクロールしても見切れないようにする（コンテナ直下先頭に配置＝sticky有効範囲を確保）。**取得は各コース/章で最大1回**（成功・失敗とも負のキャッシュ。MutationObserver から高頻度で呼ばれても再取得しない＝リクエストストーム防止）。章数の多いコースはバッチURLを分割取得。**必修以外（`type:"advanced"`＝大学受験など・`my_courses?service=basic` に載らないコース）は重い集計をせず「選択科目・卒業要件外」と明示ラベル**（実害の出た大コースを意図挙動に）。
- **/my_course に未完科目を注入**: マイコースの各「N月のレポート」の下に、その締切で**未完の科目（数学C 第1回…）と進捗**を一覧表示（`report_progresses/monthly` の締切別章内訳・免除章は除外）。どの締切に何が残っているかがページ上で一目瞭然に。取得は各月最大1回（負のキャッシュ＋再注入上限でストーム防止）。
- **視聴任意教材の扱い**: 章内の `material_type: "supplement"`（視聴任意の補助動画）は、本家の進捗カウント（`total_count`/`passed_materials`）に含まれない（HAR＋実測で確認）。拡張の動画時間・残りサマリ・教科別シェアも既定では **main のみを集計**し、本家の必修進捗と母集団を揃えている。**教科タブの設定で「視聴任意の補助動画を残り学習量に含める」に切替可**（本家準拠の完了%・予測には影響しない）。
- **サイト全体ダークモード**（自作・**実験的機能**）: 本家にダークモードは存在せず、本拡張が動的に配色を組み替えています。サイト側のデザイン変更・未知のページ構造により**表示が崩れる/一部が見づらくなる可能性があります**（その場合はトグルでライトに戻してください。学習データには一切影響しません）。filter反転でなく動的リマップ（白面→暗色/暗文字→明色）。**低彩度の文字だけ再マップし、色付きの文字（ブランド青・成功緑・警告赤/橙）は意味を保持**。スキャンは読み書き二相＋シグネチャキャッシュで軽量。本家ナビに馴染む「テーマ」トグル、`chrome.storage`で永続化。
  - **白フラッシュ対策**: `chrome.storage` は非同期で初回ペイントに間に合わないため、localStorage の同期ミラーを併用し **document_start（ペイント前）に基礎ダークを適用**（フル読み込み・教材iframeとも）。SPA遷移は暗幕フェードで隠す。
  - **テーマ切替はフェード**: 切替の瞬間だけ全要素に transition を付与しグラデーション的に遷移（`prefers-reduced-motion` では無効）。
  - **数式・数学レポート対応**: 数式レンダラ（KaTeX/MathJax/MathML）の分数線・罫線は `border-color: currentColor` で復元し境界色の一括上書きから除外。実測でZEN Studyの数式は**白背景PNG画像**のため、レポート/テスト画面では**ピクセル判定でほぼ白背景の組版画像だけソフト反転**（分数線がくっきり。写真は判定で除外され減光のまま）。
- **表示アニメーション**（一般的なアプリ風の段階表示）: 棒はベースラインから伸び、折れ線は左から描かれ、ドーナツ弧は0%→実値へ、カレンダーは列ごとにポップイン、主要数値は0からカウントアップ。予測グラフは必要ライン・不確実性帯・完了分布・完了見込みも順に現れる。**OSの「動きを減らす」設定(`prefers-reduced-motion`)では一切動かさず即確定**。表示演出のみで取得・記録には影響しない。

## 補足
- content script は `www.nnn.ed.nico` 全体にマッチ（`run_at: document_start`）。SPA遷移・React再描画に MutationObserver + history パッチで追従。/setting の本家パネルは描画前(rAF)に隠してフラッシュ防止。
- 先行事例 [Level222/zen-study-plus](https://github.com/Level222/zen-study-plus)（MIT）には API・サイト構造の理解で大いに助けられた（下記「謝辞」参照）。コードは本拡張で独自実装。機能面の焦点も異なる（あちらは動画時間表示・ショートカット等のUX改善、こちらは統計・可視化・予測）。
- 開始日/長期日別/完了日時のAPIは存在しない（`API_REFERENCE.md` 参照）ため、長期履歴は自前蓄積、予測は開始日非依存（直近ペース＋MC）。
- **日境界は 5:00 AM (JST)**: ZEN Study の日別学習数は深夜0時でなく朝5時に切り替わる。ストリーク・カレンダー・予測基点・スナップショットの日付キーはこの境界に揃える（`format.ts` の `zenTodayISO()`／端末TZ非依存）。
- 日付の "YYYY-MM-DD" は必ず `format.ts` の `parseDate`/`isoLocal` を使う（`toISOString()` はUTC基準で祝日/曜日判定がズレるため不可）。
- **計算系の統一（必修/課外の分離）**: 予測・必修判定は `learning_amounts`（合算）でなく必修の教材消化 `passed_materials` で行う。具体的に **①予測カーブの曜日重み ②必修アドバイスの現在ペース** を materialHistory の日次差分ベースに統一（非必修の曜日偏り・ペースが必修判定に混入しない）。合算に退避するのは materialHistory が皆無の導入初日のみ。
- **学習数(累計/日別)の範囲**: `learning_amounts` は**非必修コースも含む全学習**。**実績・完了見込みグラフと必修ペースは `learning_amounts` を使わず、必修の教材消化 `passed_materials`（materialHistory）のみで算出**（実績カーブ・フォールバックペース・日次サンプルすべて）。これにより非必修を大量にやっても必修の過去実績が過剰に見えない。`learning_amounts` に退避するのは materialHistory が全く無い導入初日だけ（その旨注記）。学習数・傾向（推移タブ）は従来どおり全学習の合計。**過去の実績カーブは materialHistory（導入後・日次）が薄いと短くなるため、必修コースの受験アンカー（resultLog）＋動画補間からの後方外挿を点線で重ねて導入以前も含めて復元**（教科別バーンダウンと共通の手法。「詳細ログの抽出」実行後に有効）。
- cold-start 対策の根拠（公開情報）: ベイズ平均/経験ベイズ縮小（IMDb加重平均・平均への回帰）、事後予測分布による小標本の予測区間（母数不確実性=epistemic ＋ 日々のばらつき=aleatoric を両方含める。母数不確実性を無視した区間は小標本でも広がらない）。
